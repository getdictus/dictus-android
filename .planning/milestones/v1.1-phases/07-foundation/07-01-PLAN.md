**OnboardingViewModelTest.kt (DEBT-03):**
Fix the test `advanceStep from step 6 writes HAS_COMPLETED_ONBOARDING to DataStore` (starting at line 180). The OnboardingViewModel.advanceStep() logic is:
- `step == 5 && !download` -> return (blocked)
- `step == 7` -> completeOnboarding()
- `step < 7` -> step + 1

So reaching step 6 and calling advanceStep() once moves to step 7 (via `step < 7` branch). A SECOND advanceStep() call when step==7 triggers completeOnboarding() which writes HAS_COMPLETED_ONBOARDING.

The current test calls advanceStep() only once after reaching step 6 (line 190), then asserts. Fix by adding a second advanceStep() + advanceUntilIdle() call:

Replace lines 189-196 with:
```kotlin
            // Tap "Commencer" on step 6 -> moves to step 7
            viewModel.advanceStep()
            // Step 7 triggers completeOnboarding()
            viewModel.advanceStep()
            advanceUntilIdle()

            assertTrue(
                "HAS_COMPLETED_ONBOARDING should be true",
                fakeDataStore.editedPrefs[PreferenceKeys.HAS_COMPLETED_ONBOARDING] == true
            )
```

Also update the comment on line 113 of OnboardingViewModel.kt if it says "Step 6: triggers DataStore persistence" -- it should say "Step 7" since `step == 7` is the actual trigger. This is a doc-only fix.
