    Make 4 targeted changes to DictusImeService.kt:

    **Change 1 — Import swap:**
    Replace:
    ```kotlin
    import dev.pivisolutions.dictus.ime.suggestion.StubSuggestionEngine
    ```
    With:
    ```kotlin
    import dev.pivisolutions.dictus.ime.suggestion.DictionaryEngine
    ```

    **Change 2 — Engine initialization (line ~87):**
    Replace:
    ```kotlin
    private val suggestionEngine: SuggestionEngine = StubSuggestionEngine()
    ```
    With:
    ```kotlin
    // Production suggestion engine: loads AOSP FR/EN dictionary from assets on
    // Dispatchers.IO, performs accent-insensitive prefix matching with frequency
    // ranking, and boosts personal dictionary words. Until dictionary loads
    // (~500ms), returns empty suggestions gracefully.
    private val suggestionEngine: SuggestionEngine by lazy {
        DictionaryEngine(
            context = applicationContext,
            dataStore = entryPoint.dataStore(),
            coroutineScope = bindingScope,
        )
    }
    ```
    WHY `by lazy`: The `entryPoint` and `applicationContext` are not available at field-init time in an InputMethodService. `lazy` defers initialization to first access (which happens in `onUpdateSelection` after `onCreate`).

    **Change 3 — Personal dictionary counting in onSuggestionSelected (line ~297):**
    In the `onSuggestionSelected` lambda, AFTER the existing `ic.commitText("$suggestion ", 1)` line, add:
    ```kotlin
    // Count this suggestion selection toward the personal dictionary
    // learning threshold (2 types = learned). The engine's personalDictionary
    // is accessed via cast since we know the runtime type is DictionaryEngine.
    (suggestionEngine as? DictionaryEngine)?.personalDictionary?.recordWordTyped(suggestion)
    ```

    Also in the `onCurrentWordSelected` lambda (line ~308), BEFORE the `if (word.isNotEmpty())` block, add personal dict counting for the raw typed word:
    ```kotlin
    // Count the committed raw word toward personal dictionary
    if (word.isNotEmpty()) {
        (suggestionEngine as? DictionaryEngine)?.personalDictionary?.recordWordTyped(word)
        ic.commitText(" ", 1)
        _suggestions.value = emptyList()
        _currentWord.value = ""
    }
    ```

    **Change 4 — Post-dictation suggestion clear (in the onConfirm lambda, line ~354-359):**
    After the existing `commitText(text)` call inside the `onConfirm` lambda in RecordingScreen, add:
    ```kotlin
    // Clear suggestions after voice transcription so the bar
    // does not show stale suggestions from the last typed word.
    // Suggestions resume when user types on keyboard.
    _suggestions.value = emptyList()
    _currentWord.value = ""
    ```
    This goes inside the `if (text != null)` block, after `commitText(text)` and the Timber.d log line.

    **No other changes.** The rest of the wiring (onUpdateSelection calling getSuggestions, SuggestionBar rendering, etc.) is already in place from Phase 5.
