## Integration Gaps

### 1. KEYBOARD_LAYOUT not consumed by IME (KBD-02)

- **Written by:** `SettingsViewModel.setKeyboardLayout()`, `OnboardingViewModel.completeOnboarding()`
- **Not read by:** `DictusImeService.KeyboardContent()` — `KeyboardScreen.kt:58` hardcodes `"azerty"`
- **Impact:** User can toggle AZERTY/QWERTY in Settings but IME always shows AZERTY
- **Severity:** Non-blocking — AZERTY is the correct default for French users, and the toggle is a secondary feature

### 2. POST_NOTIFICATIONS missing from AndroidManifest (DICT-08)

- **Requested by:** `PermissionHelper.kt` at runtime on API 33+
- **Missing from:** `app/src/main/AndroidManifest.xml`
- **Impact:** Foreground service notification may not show on API 33+ devices
- **Severity:** Non-blocking — Pixel 4 runs API 33 max, foreground service works regardless

## Broken Flow

### Settings keyboard layout toggle -> IME layout change

- User opens Settings -> taps AZERTY/QWERTY picker -> SettingsViewModel writes to DataStore -> **IME never reads it**
- Fix: Read `KEYBOARD_LAYOUT` in `DictusImeService.KeyboardContent()` (same pattern as THEME, HAPTICS_ENABLED)
