**OnboardingViewModel.kt changes:**

1. Add `SavedStateHandle` as third constructor parameter (Hilt auto-provides it for @HiltViewModel):
   ```kotlin
   @HiltViewModel
   class OnboardingViewModel @Inject constructor(
       private val dataStore: DataStore<Preferences>,
       private val modelDownloader: ModelDownloader,
       private val savedStateHandle: SavedStateHandle,
   ) : ViewModel()
   ```

2. Replace the three volatile MutableStateFlow fields with SavedStateHandle-backed versions:
   - `_currentStep`: Use `savedStateHandle.getStateFlow("currentStep", 1)` for the public read. For writes, use `savedStateHandle["currentStep"] = value`. Keep a private getter/setter helper.
   - `_micPermissionGranted`: Use `savedStateHandle.getStateFlow("micPermissionGranted", false)`. Writes via `savedStateHandle["micPermissionGranted"] = value`.
   - `_imeActivated`: Use `savedStateHandle.getStateFlow("imeActivated", false)`. Writes via `savedStateHandle["imeActivated"] = value`.

3. Update `advanceStep()`, `goBack()`, `setMicPermissionGranted()`, `setImeActivated()` to write through SavedStateHandle instead of MutableStateFlow.value.

4. Keep `_selectedLayout`, `_downloadProgress`, `_modelDownloadComplete`, `_downloadError` as regular MutableStateFlow — these are transient and should reset on process death (download restarts, layout re-chosen).

**Important:** `savedStateHandle.getStateFlow()` returns a read-only `StateFlow`, so the public `val currentStep: StateFlow<Int>` signature stays the same. No callers change.
