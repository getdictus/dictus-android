# Phase 4 — UI Design Contract

> Visual and interaction contract for Phase 4: Model Management + Onboarding.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker. Updated from Pencil mockups.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (native Android — shadcn not applicable) |
| Preset | not applicable |
| Component library | Jetpack Compose + Material 3 (`androidx.compose.material3`) |
| Icon library | Lucide icons (matching iOS parity) — fallback: Material Icons Extended |
| Font | **DM Sans** (per Pencil mockups — matches iOS app) |

**Note:** This is a Kotlin/Compose project. The shadcn gate does not apply. The design system is
established in `core/src/main/java/dev/pivisolutions/dictus/core/theme/`. The existing codebase uses
Roboto (system default) — **migrate to DM Sans** to match iOS parity and Pencil mockups. DM Sans is
available on Google Fonts and can be bundled as a resource font.

---

## Spacing Scale

Declared values (multiples of 4dp — Android uses dp, not px):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4dp | Icon gaps, inline padding, divider insets, model-info gap between name+subtitle |
| sm | 8dp | Compact element spacing, chip padding, progress dot gap |
| md | 16dp | Default element spacing, card content padding, mode picker gap, settings row padding |
| lg | 24dp | Section gaps between icon→text, text→card, content groups; step-dots top padding |
| xl | 32dp | Onboarding horizontal padding, gap between waveform and text block (welcome) |
| 2xl | 48dp | Major section breaks |
| 3xl | 64dp | Page-level spacing |

Exceptions:
- Touch targets for all tappable elements: minimum 48dp (Android accessibility minimum)
- Bottom navigation bar height: 80dp (pill nav per mockup frame `d7cJl`)
- GlassCard corner radius: 16dp (per mockup frame `eKShU`) — not a spacing token
- CTA button corner radius: 14dp (per mockups)
- CTA button height: 56dp (per mockups)
- Onboarding progress dots: 8dp diameter, 8dp gap between dots
- Onboarding bottom padding (below dots): 60dp
- Model card icon background: 48dp × 48dp, corner 12dp
- Fake settings card row padding: 14dp vertical, 16dp horizontal
- Toggle dimensions: 51dp × 31dp, corner 16dp
- Success check circle: 120dp outer, 56dp icon inside, corner 60dp (full circle), stroke 3dp

## Typography

Source font: **DM Sans** (per Pencil mockups — iOS parity).
Current codebase uses Roboto — update `DictusTypography.kt` to use DM Sans.

| Role | Size | Weight | Line Height | Letter Spacing | Usage |

## Interaction Contracts

### Onboarding Step Advancement

- Each step validates its condition before enabling the CTA
- Conditions:
  - Step 2 (mic): CTA enabled only after permission granted OR already granted
  - Step 3 (keyboard): CTA shows "Ouvrir les Réglages" until IME is active, then switches to "Continuer"
  - Step 4 (mode): CTA always enabled (default selection = ABC)
  - Step 5 (model): CTA changes from "Télécharger" → disabled "Téléchargement en cours..." → "Continuer"
  - Step 6 (success): CTA always enabled ("Commencer")
- No back navigation on Step 1
- Back navigation on Steps 2–6: goes to previous step without resetting validated state

### Model Download

- Download starts immediately on tap of "Télécharger"
- CTA replaced by disabled state: `#161C2C` fill, `#2A2A2E` border, "Téléchargement en cours..." `#6B6B70`
- Progress shown inside model card: bar + percentage label
- Progress updates via `Flow<Int>` (percentage 0–100) from `ModelDownloader`
- On error: progress bar replaced by "Échec du téléchargement." + "Réessayer" tappable text (`#EF4444`) — same card, no dialog
- On cancel: not supported in MVP
- On completion: CTA becomes "Continuer" (accent gradient)

### Model Deletion

- "Supprimer" button shown only on downloaded models
- Tap triggers a bottom sheet confirmation (not a dialog):
  - Title: "Supprimer [model name] ?"
  - Body: "Ce modèle sera supprimé de votre appareil."
  - Confirm button: "Supprimer" (`#EF4444` text, full-width)
  - Cancel button: "Annuler" (`#6B6B70` text)
- Last model guard: if only one model downloaded, "Supprimer" button is hidden

### Active Model Selection (Settings)

- Tap "Modèle actif" row opens a bottom sheet with a radio list of downloaded models
- Selected model: radio filled `#3D7EFF`
- Persisted to DataStore on selection
- Bottom sheet title: "Choisir un modèle"
- Empty state: row disabled, secondary text "Aucun modèle téléchargé" (`#6B6B70`)

### Navigation

- Onboarding shown when `HAS_COMPLETED_ONBOARDING = false` in DataStore
- On completion (step 6 CTA "Commencer"), write `HAS_COMPLETED_ONBOARDING = true`, navigate to Home tab
- Bottom nav bar always visible on main 3-tab layout
- No back stack between tabs (each tab retains its own scroll position)
- Back press from any main tab: minimizes app (Android default behavior)

### Press Animation (GlassCard and Buttons)

- Press: scale to 0.96 + opacity to 0.85
- Spring animation: `dampingRatio = 0.6`, `stiffness = Spring.StiffnessMedium`
- Applied to: model cards, option cards, setting rows (tappable), onboarding CTA buttons

## Loading and Skeleton States

- Model list loading: 2 skeleton GlassCards (animated shimmer `#FFFFFF08` to `#FFFFFF15`)
- Active model loading on Home tab: skeleton card (same shimmer pattern)
- Settings load from DataStore: no skeleton — use last cached value, update when ready
- Download progress: live progress bar (no spinner)

Shimmer animation: horizontal sweep left-to-right, 1200ms duration, repeat infinite.

## Error States

| Scenario | UI Treatment |
|----------|-------------|
| Model download network failure | Inline in model card: "Échec du téléchargement." + "Réessayer" tappable text (`#EF4444`) |
| Mic permission permanently denied (step 2) | Replace CTA with "Ouvrir les Réglages" button + explanatory text "Autorisez le micro dans les réglages système pour continuer." |
| IME never activated (step 3) | Persistent "Ouvrir les Réglages" button — user must return from Settings for check to run |
| Log export failure | Toast: "Impossible d'exporter les logs." (2 seconds) |
| Generic DataStore read failure | Settings rows show "—" as value, not blank |
