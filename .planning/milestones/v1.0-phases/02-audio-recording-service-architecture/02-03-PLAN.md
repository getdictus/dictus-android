The accent popup in KeyButton.kt (lines 278-295) uses `Alignment.TopCenter` with `IntOffset(0, -100)` and no horizontal clamping. For leftmost keys like "a", the popup (6 accents x 44dp = 264dp wide) overflows the left screen edge. The `resolveAccentIndex()` function (lines 102-110) has the same centered assumption.

**Fix the popup positioning:**

1. Add a `keyPositionXPx` state variable (Float, initialized to 0f) alongside the existing `keyWidthPx` state.

2. In the `Box` modifier chain, add `onGloballyPositioned` to capture the key's absolute X position on screen:
```kotlin
.onGloballyPositioned { coordinates ->
    val positionInWindow = coordinates.positionInWindow()
    keyPositionXPx = positionInWindow.x
}
```
Note: `onSizeChanged` already exists at line 237. Keep it for `keyWidthPx`. Add `onGloballyPositioned` right after it. Import `androidx.compose.ui.layout.onGloballyPositioned` and `androidx.compose.ui.layout.positionInWindow`.

3. Compute a clamped horizontal offset for the accent popup. Before the accent Popup block (line 278), calculate:
```kotlin
val popupWidthPx = (accentChars?.size ?: 0) * accentCellWidthPx
val centeredOffsetPx = (keyWidthPx - popupWidthPx) / 2f
val absolutePopupLeftPx = keyPositionXPx + centeredOffsetPx
val screenWidthPx = with(density) { /* get from LocalConfiguration or view.width */ }
```

Use `LocalConfiguration.current.screenWidthDp` with density to get screen width in px, or simpler: use `view.rootView.width` (the view is already available via `LocalView.current`).

Clamp:
```kotlin
val clampedOffsetPx = when {
    absolutePopupLeftPx < 0f -> centeredOffsetPx - absolutePopupLeftPx  // shift right
    absolutePopupLeftPx + popupWidthPx > screenWidthPx -> centeredOffsetPx - (absolutePopupLeftPx + popupWidthPx - screenWidthPx)  // shift left
    else -> centeredOffsetPx  // no clamping needed
}
val clampedOffsetDp = with(density) { clampedOffsetPx.toInt() }
```

4. Apply the clamped offset to the accent Popup:
```kotlin
Popup(
    alignment = Alignment.TopStart,  // Changed from TopCenter to TopStart
    offset = IntOffset(clampedOffsetDp, with(density) { (-100).dp.roundToPx() }),
    ...
)
```
Wait -- Popup offset is in px already (IntOffset). Use `clampedOffsetPx.toInt()` directly.

Actually, Compose Popup `offset` uses IntOffset in pixels. And `Alignment.TopCenter` already centers relative to the anchor. So the cleanest approach:

- Keep `Alignment.TopCenter`
- Compute the HORIZONTAL SHIFT needed to clamp, and pass it as the x component of IntOffset
- The shift is: `clampedOffsetPx - centeredOffsetPx` (the delta from the natural center)

Let me simplify:
```kotlin
// Natural centered position of popup left edge in screen coordinates
val popupWidthPx = (accentChars?.size ?: 0) * accentCellWidthPx
val naturalPopupLeftPx = keyPositionXPx + (keyWidthPx - popupWidthPx) / 2f
val screenWidthPx = view.rootView.width.toFloat()

// Clamp to [0, screenWidth - popupWidth]
val clampedLeftPx = naturalPopupLeftPx.coerceIn(0f, (screenWidthPx - popupWidthPx).coerceAtLeast(0f))
val shiftPx = (clampedLeftPx - naturalPopupLeftPx).toInt()
```

Then in the Popup:
```kotlin
offset = IntOffset(shiftPx, -100.dp.roundToPx())  // wait, -100 is already in px? No.
```

Looking at the existing code: `IntOffset(0, -100)` -- this -100 is in PIXELS (IntOffset is always px). Keep the same y offset value.

```kotlin
offset = IntOffset(shiftPx, -100)
```

5. **Fix `resolveAccentIndex()` to use the same clamped offset:**

The function currently computes `popupLeftPx = (keyWidthPx - popupWidthPx) / 2f` which is the centered offset relative to the key. With clamping, the popup is shifted. Update it to account for the shift:

```kotlin
fun resolveAccentIndex(pointerX: Float): Int? {
    val accents = accentChars ?: return null
    if (accents.isEmpty() || keyWidthPx <= 0) return null

    val popupWidthPx = accents.size * accentCellWidthPx
    val naturalPopupLeftPx = (keyWidthPx - popupWidthPx) / 2f
    val screenWidthPx = view.rootView.width.toFloat()
    val naturalAbsLeftPx = keyPositionXPx + naturalPopupLeftPx
    val clampedAbsLeftPx = naturalAbsLeftPx.coerceIn(0f, (screenWidthPx - popupWidthPx).coerceAtLeast(0f))
    val shiftPx = clampedAbsLeftPx - naturalAbsLeftPx
    val adjustedPopupLeftPx = naturalPopupLeftPx + shiftPx

    val index = ((pointerX - adjustedPopupLeftPx) / accentCellWidthPx).toInt()
    return index.takeIf { it in accents.indices }
}
```

**DRY optimization:** Extract the clamping math into a local helper or compute `shiftPx` once as a derived value (using `remember`) so both the Popup offset and `resolveAccentIndex` use the same value. A simple approach: compute `accentShiftPx` as a remembered derivation outside the gesture/popup blocks:

```kotlin
val accentShiftPx = remember(keyWidthPx, keyPositionXPx, accentChars) {
    val accents = accentChars ?: return@remember 0
    if (accents.isEmpty() || keyWidthPx <= 0) return@remember 0
    val popupWidthPx = accents.size * accentCellWidthPx
    val naturalLeftPx = keyPositionXPx + (keyWidthPx - popupWidthPx) / 2f
    val screenWidthPx = view.rootView.width.toFloat()
    val clampedLeftPx = naturalLeftPx.coerceIn(0f, (screenWidthPx - popupWidthPx).coerceAtLeast(0f))
    (clampedLeftPx - naturalLeftPx).toInt()
}
```

Then use `accentShiftPx` in both the Popup offset and `resolveAccentIndex`.
