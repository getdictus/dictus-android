  <action>
Create the Compose keyboard UI. The keyboard is structured as:

```
KeyboardScreen (root, manages state)
+-- KeyboardView (renders rows based on current layer)
|   +-- KeyRow (horizontal row of keys)
|   |   +-- KeyButton (individual key with tap/long-press)
|   |   +-- AccentPopup (overlay when long-pressing accent key)
|   +-- ... more rows
+-- MicButtonRow (placeholder below keyboard)
```

**KeyboardScreen.kt** -- Root composable that manages keyboard state:
- State: `currentLayer: KeyboardLayer` (LETTERS, NUMBERS, SYMBOLS), `isShifted: Boolean`, `isCapsLock: Boolean`, `currentLayout: String` ("azerty" default), `showAccentPopup: Boolean`, `accentKey: Char?`, `accentPosition: Offset`
- Callbacks received from DictusImeService: `onCommitText: (String) -> Unit`, `onDeleteBackward: () -> Unit`, `onSendReturn: () -> Unit`
- Height: 280.dp for keyboard + 56.dp for mic row = 336.dp total
- Wraps content in DictusTheme
- Handles key press logic:
  - CHARACTER: if shifted, commit uppercase, then turn off shift (unless caps lock). Call onCommitText(output).
  - SHIFT: single tap toggles isShifted, double-tap toggles isCapsLock
  - DELETE: call onDeleteBackward. Support key repeat: while held, delete every 50ms after initial 400ms delay.
  - SPACE: onCommitText(" ")
  - RETURN: onSendReturn()
  - LAYER_SWITCH: toggle between LETTERS and NUMBERS. If label is "#+=", switch to SYMBOLS. If "123", switch to NUMBERS. If "ABC", switch to LETTERS.
  - EMOJI: no-op in Phase 1 (log "Emoji picker not yet implemented")
  - MIC: no-op in Phase 1 (log "Mic not yet implemented")
  - ACCENT_ADAPTIVE: show accent popup for the last typed character
- For long-press on CHARACTER keys: check AccentMap.hasAccents(char), if true show AccentPopup

**KeyboardView.kt** -- Renders the correct rows for the current layer:
- Takes `layer: KeyboardLayer`, `isShifted: Boolean`, `layout: String`, `onKeyPress: (KeyDefinition) -> Unit`, `onKeyLongPress: (KeyDefinition, Offset) -> Unit`
- Determines which rows to render:
  - LETTERS: `KeyboardLayouts.lettersForLayout(layout)` -- apply shift to labels if isShifted
  - NUMBERS: `KeyboardLayouts.numbersRows`
  - SYMBOLS: `KeyboardLayouts.symbolsRows`
- Column of KeyRow composables
- Background: DictusColors.Background
- Vertical padding: 4.dp top, 2.dp bottom

**KeyRow.kt** -- Horizontal row of keys:
- Takes `keys: List<KeyDefinition>`, `isShifted: Boolean`, callbacks
- Row with horizontal arrangement, each key gets weight based on widthMultiplier
- Horizontal padding: 2.dp on sides
- Horizontal spacing between keys: 4.dp

**KeyButton.kt** -- Individual key rendering:
- Takes `key: KeyDefinition`, `isShifted: Boolean`, `onPress: () -> Unit`, `onLongPress: ((Offset) -> Unit)?`
- Visual style:
  - Background: `DictusColors.KeyBackground` for CHARACTER/SPACE, `DictusColors.KeySpecialBackground` for SHIFT/DELETE/LAYER_SWITCH/RETURN/EMOJI/MIC
  - Text color: `DictusColors.KeyText`
  - Corner radius: 8.dp (RoundedCornerShape)
  - Height: 46.dp for regular keys
  - Vertical margin: 3.dp
  - Shadow/elevation: 1.dp for subtle depth
- Text rendering:
  - CHARACTER: show `key.label` (uppercase if shifted), font size 20.sp
  - SHIFT: show up-arrow icon, highlighted if isShifted (DictusColors.Accent background)
  - DELETE: show backspace icon
  - SPACE: show "space" text, 14.sp
  - RETURN: show return icon
  - LAYER_SWITCH: show key.label ("?123" or "ABC"), 14.sp
- Gesture handling:
  - Use `Modifier.pointerInput` with `detectTapGestures(onTap = { onPress() }, onLongPress = { offset -> onLongPress?.invoke(offset) })`
  - For DELETE key: use separate logic with `awaitPointerEventScope` for key repeat. On press start repeating after 400ms delay, every 50ms.
  - Do NOT use `Modifier.clickable` together with `pointerInput` (conflict per research pitfall 7)

**AccentPopup.kt** -- Popup for accent selection:
- Takes `char: Char`, `position: Offset`, `onAccentSelected: (String) -> Unit`, `onDismiss: () -> Unit`
- Shows a horizontal row of accent options from AccentMap.accentsFor(char)
- Background: DictusColors.Surface with 12.dp corner radius
- Each accent option: 40.dp x 40.dp box with 20.sp text
- Positioned above the key that was long-pressed (use Popup composable)
- Dismiss on tap outside or accent selection

**MicButtonRow.kt** -- Placeholder row below keyboard:
- Height: 56.dp
- Background: DictusColors.Background
- Center: mic icon button (use a simple circle with mic text/icon), DictusColors.Accent
- Left: keyboard switcher icon (triggers InputMethodManager.showInputMethodPicker)
- Non-functional mic in Phase 1 -- just log "Mic pressed (not yet implemented)"
- For keyboard switcher: the DictusImeService should call `switchInputMethod()` or pass a lambda

@Composable
override fun KeyboardContent() {

        },
    )
}
