## Spacing Scale

Declared values (multiples of 4dp — Android uses dp, not px):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4dp | Icon gaps, inline padding, divider insets |
| sm | 8dp | Compact element spacing, chip padding |
| md | 16dp | Default element spacing, card content padding |
| lg | 24dp | Section padding, page horizontal margins |
| xl | 32dp | Layout gaps between major sections |
| 2xl | 48dp | Major section breaks (e.g. onboarding page top padding) |
| 3xl | 64dp | Page-level spacing (e.g. bottom of onboarding before CTA) |

Exceptions:
- Touch targets for all tappable elements: minimum 48dp (Android accessibility minimum)
- Bottom navigation bar height: 80dp (pill nav per mockup frame `d7cJl`)
- GlassCard corner radius: 16dp (per mockup frame `eKShU`) — not a spacing token
- Onboarding progress dot size: 8dp diameter, 6dp gap between dots

## Typography

Source: `core/src/main/java/dev/pivisolutions/dictus/core/theme/DictusTypography.kt`
Design reference also specifies Inter — note that existing code uses system default (Roboto).
Use system default (Roboto) consistently with current implementation.

| Role | Size | Weight | Line Height | Material 3 Style | Usage |
|------|------|--------|-------------|-----------------|-------|
| Display | 28sp | Bold (700) | 1.2 (34sp) | `headlineLarge` | Onboarding step titles, success screen heading |
| Heading | 22sp | Bold (700) | 28sp | `titleLarge` | Screen titles, tab section headers — source: `DictusTypography.kt` |
| Body | 16sp | Normal (400) | 24sp (1.5) | `bodyLarge` | Card body text, descriptions, settings values — source: `DictusTypography.kt` |
| Label | 12sp | Medium (500) | 16sp | `labelMedium` | Model badges, storage indicators, nav tab labels |

**Weights used: Normal (400) + Medium (500) + Bold (700).**
Note: Three weights are used here because the existing codebase already defines labelSmall at 500
and titleLarge at Bold. Medium is reserved for labels only; do not use it for body or headings.

## Interaction Contracts

### Onboarding Step Advancement

- Each step validates its condition before enabling the CTA
- Conditions:
  - Step 2 (mic): CTA enabled only after permission granted OR already granted
  - Step 4 (IME): CTA enabled only when IME is active (checked via `InputMethodManager`, same as `ImeStatusCard`)
  - Step 5 (model): CTA enabled only when download completes (100%)
  - Step 6 (test): CTA label changes to "Terminer" only after at least one successful transcription
- Steps 1, 3: CTA always enabled
- No back navigation on Step 1 (no back button shown)
- Back navigation on Steps 2–6: goes to previous step without resetting validated state

### Model Download

- Download starts immediately on tap of "Télécharger"
- Button replaced by progress bar + percentage label during download
- Progress updates via `Flow<Int>` (percentage 0–100) from `ModelDownloader`
- On error: progress bar replaced by "Échec du téléchargement. Réessayer." text link (`#EF4444`) — same card, no dialog
- On cancel: not supported in MVP (user must wait or kill app)
- On completion: card transitions to "downloaded" state with "Supprimer" button

### Model Deletion

- "Supprimer" button shown only on downloaded models
- Tap triggers a bottom sheet confirmation (not a dialog):
  - Title: "Supprimer [model name] ?"
  - Body: "Ce modèle sera supprimé de votre appareil."
  - Confirm button: "Supprimer" (`#EF4444` text, full-width)
  - Cancel button: "Annuler" (`#6B6B70` text)
- Last model guard: if only one model downloaded, "Supprimer" button is hidden (not disabled, not present)

### Active Model Selection (Settings)

- Tap "Modèle actif" row opens a bottom sheet with a radio list of downloaded models
- Selected model: radio filled `#3D7EFF`
- Persisted to DataStore on selection
- Bottom sheet title: "Choisir un modèle"
- Empty state (no models downloaded): row disabled, secondary text "Aucun modèle téléchargé" (`#6B6B70`)

### Navigation

- Onboarding shown when `HAS_COMPLETED_ONBOARDING = false` in DataStore
- On completion, write `HAS_COMPLETED_ONBOARDING = true`, navigate to Home tab
- Bottom nav bar always visible on main 3-tab layout
- No back stack between tabs (each tab retains its own scroll position)
- Back press from any main tab: minimizes app (Android default behavior)

### Press Animation (GlassCard and Buttons)

- Press: scale to 0.96 + opacity to 0.85
- Spring animation: `dampingRatio = 0.6`, `stiffness = Spring.StiffnessMedium`
- Applied to: model cards, setting rows (tappable), onboarding CTA buttons

## Loading and Skeleton States

Per "Claude's Discretion" in `04-CONTEXT.md`:

- Model list loading: 2 skeleton GlassCards (animated shimmer `#FFFFFF08` to `#FFFFFF15`)
- Active model loading on Home tab: skeleton card (same shimmer pattern)
- Settings load from DataStore: no skeleton — use last cached value, update when ready
- Download progress: live progress bar (no spinner)

Shimmer animation: horizontal sweep left-to-right, 1200ms duration, repeat infinite.

## Error States

Per "Claude's Discretion" in `04-CONTEXT.md`:

| Scenario | UI Treatment |
|----------|-------------|
| Model download network failure | Inline in model card: "Échec du téléchargement." + "Réessayer" tappable text (`#EF4444`) |
| Mic permission permanently denied (onboarding step 2) | Replace CTA with "Ouvrir les Réglages" button + explanatory text "Autorisez le micro dans les réglages système pour continuer." |
| IME never activated (onboarding step 4) | Persistent "Ouvrir les Réglages" button — user must return from Settings for check to run |
| Log export failure | Toast: "Impossible d'exporter les logs." (2 seconds) |
| Generic DataStore read failure | Settings rows show "--" as value, not blank |
