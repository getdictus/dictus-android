Create `.github/workflows/ci.yml` with:
- `name: CI`
- Triggers: `on: push: branches: [main]` and `on: pull_request: branches: [main]`
- Two parallel jobs:
  - `lint` job: `runs-on: ubuntu-latest`, steps: `actions/checkout@v4`, `actions/setup-java@v4` (distribution: temurin, java-version: '17'), `gradle/actions/setup-gradle@v4`, run `./gradlew lint`
  - `test` job: `runs-on: ubuntu-latest`, steps: same setup, run `./gradlew test`, then `./gradlew licenseeDebug assembleDebug` (IMPORTANT: licenseeDebug MUST run before or together with assembleDebug — the licensee plugin generates artifacts.json into assets, but it is NOT wired into the assemble task, only into check. Without licenseeDebug, CI-built APKs will have no license data and LicencesScreen shows an empty list. See RESEARCH.md Pitfall 5.), then `actions/upload-artifact@v4` uploading `app/build/outputs/apk/debug/app-debug.apk` as artifact name `debug-apk`

Create `.github/workflows/release.yml` with:
- `name: Release`
- Trigger: `on: push: tags: ['v*']`
- `permissions: contents: write`
- Single job `release` on `ubuntu-latest`:
  - `actions/checkout@v4`
  - `actions/setup-java@v4` (temurin, 17)
  - `gradle/actions/setup-gradle@v4`
  - "Decode keystore" step: `echo "${{ secrets.SIGNING_KEYSTORE }}" | base64 -d > ${{ github.workspace }}/release.jks`
  - "Extract version from tag" step with `id: version`:
    ```
    TAG="${{ github.ref_name }}"
    VERSION="${TAG#v}"
    MAJOR=$(echo $VERSION | cut -d. -f1)
    MINOR=$(echo $VERSION | cut -d. -f2)
    PATCH=$(echo $VERSION | cut -d. -f3)
    VERSION_CODE=$(( MAJOR * 10000 + MINOR * 100 + PATCH ))
    echo "version_name=$VERSION" >> $GITHUB_OUTPUT
    echo "version_code=$VERSION_CODE" >> $GITHUB_OUTPUT
    ```
  - "Build signed release APK" step with env vars: `SIGNING_KEYSTORE_PATH: ${{ github.workspace }}/release.jks`, `SIGNING_KEY_ALIAS: ${{ secrets.SIGNING_KEY_ALIAS }}`, `SIGNING_KEY_PASSWORD: ${{ secrets.SIGNING_KEY_PASSWORD }}`, `SIGNING_STORE_PASSWORD: ${{ secrets.SIGNING_STORE_PASSWORD }}`, `VERSION_CODE: ${{ steps.version.outputs.version_code }}`, `VERSION_NAME: ${{ steps.version.outputs.version_name }}`, run `./gradlew licenseeRelease assembleRelease` (IMPORTANT: licenseeRelease MUST precede assembleRelease so artifacts.json is generated into assets before the APK is packaged. See RESEARCH.md Pitfall 5.)
  - "Upload to GitHub Releases" step: `softprops/action-gh-release@v2`, `name: Dictus ${{ steps.version.outputs.version_name }}`, `files: app/build/outputs/apk/release/app-release.apk`
  </action>

In `app/build.gradle.kts`, inside the `android {}` block, add a `signingConfigs` block BEFORE `defaultConfig`:

```kotlin
signingConfigs {
    create("release") {
        val keystorePath = System.getenv("SIGNING_KEYSTORE_PATH")
        val keyAlias = System.getenv("SIGNING_KEY_ALIAS")
        val keyPassword = System.getenv("SIGNING_KEY_PASSWORD")
        val storePassword = System.getenv("SIGNING_STORE_PASSWORD")
        if (keystorePath != null) {
            storeFile = file(keystorePath)
            this.storePassword = storePassword
            this.keyAlias = keyAlias
            this.keyPassword = keyPassword
        }
    }
}
```

In `defaultConfig`, add CI version override AFTER the existing `versionName = "1.0.0"` line:

```kotlin
val ciVersionCode = System.getenv("VERSION_CODE")?.toIntOrNull()
val ciVersionName = System.getenv("VERSION_NAME")
if (ciVersionCode != null) versionCode = ciVersionCode
if (ciVersionName != null) versionName = ciVersionName
```

Add a `buildTypes` block after `defaultConfig` (or inside existing android block if buildTypes exists):

```kotlin
buildTypes {
    release {
        signingConfig = signingConfigs.getByName("release")
        isMinifyEnabled = false
    }
}
```

```
keystore.properties
```
